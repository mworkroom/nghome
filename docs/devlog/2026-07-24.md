# 2026-07-24 NANGOK 홈 개발 로그

## Work Summary

안내 상세 페이지의 인라인 CSS를 기존 공통 `style.css`로 통합했다. 공통 스타일의 기준을 하나로 유지하면서도 메인 화면에 영향을 주지 않도록 안내 페이지 범위를 분리했다.

## Work Completed

* `guides/pdf.html`, `guides/ai.html`에서 인라인 `<style>` 블록을 제거하고 `../style.css`를 연결했다.
* 안내 페이지 공통 레이아웃, 헤더, 뒤로가기 버튼, 그림 카드, 반응형 규칙을 `style.css`로 옮겼다.
* AI 안내에만 필요한 프롬프트 카드, 복사 버튼, 텍스트 입력창 스타일은 `guide-page--ai` 범위로 한정했다.
* PDF와 AI 페이지의 기존 모바일 전환 폭(각각 560px, 680px)을 유지했다.
* 인라인 CSS 제거 여부와 `git diff --check`로 공백 오류가 없는 것을 확인했다.

## Decisions

새 `guide.css`를 만들지 않고 기존 `style.css`를 공통 스타일의 단일 기준으로 사용했다. 안내 페이지에 `guide-page`, `guide-page--pdf`, `guide-page--ai` 클래스를 부여해 같은 파일 안에서도 랜딩 페이지 CSS와 충돌하지 않게 했다.

## Current Status

PDF 및 AI 안내 페이지는 외부 `style.css`만 사용하며, 공통·전용 스타일의 범위가 분리되어 있다. 정적 파일 구조와 CSS 공백 오류를 확인했다.

## Next Steps

* 실제 배포 화면에서 데스크톱과 모바일 폭의 시각적 확인이 필요하면 별도로 점검한다.
